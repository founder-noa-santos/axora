# Explore a Repository Efficiently

### Step 1: Start from entrypoints and public interfaces
**Validation:** Locate binaries, server startup, and exported modules first.

### Step 2: Use fast search before deep reads
**Validation:** Identify exact files and symbols before opening long files.

### Step 3: Map the runtime path, not every module
**Validation:** Focus on the code that actually executes for the task.
