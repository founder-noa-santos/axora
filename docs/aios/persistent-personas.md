# Persistent Personas

Persistent personas are durable stewardship identities scoped per workspace.

## Core personas

- `Backlog Steward`
- `Planning Steward`
- `Architecture Steward`
- `Implementation Steward`
- `Review Steward`
- `Verification Steward`
- `Reliability Steward`
- `Knowledge Steward`

## Rules

- Personas persist via stable `persona_id` records in canonical work-management state.
- Persona assignments can target stories, prepared stories, requirements, work items, or workspace-level responsibilities.
- Ephemeral workers remain valid, but a persona owns the outcome, claims, and memory promotion path.
- Planning, Verification, and relevant domain stewardship must discharge profile-required gates before closure.
