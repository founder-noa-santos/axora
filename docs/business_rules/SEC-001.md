# Input Validation

## Rule Definition

All user input must be validated and sanitized before processing to prevent injection attacks and data corruption.

## Validation Criteria

1. **Type Validation** — Input must match expected type (string, number, boolean, etc.)
2. **Length Validation** — Input must be within acceptable length bounds
3. **Format Validation** — Input must match expected format (email, URL, phone, etc.)
4. **Range Validation** — Numeric input must be within acceptable range
5. **Allowlist Validation** — Input must be from acceptable set of values

## Implementation Notes

- Use `validator` crate for declarative validation
- Validate on API boundary (handlers), not just in models
- Return specific error messages for validation failures
- Log validation failures for security monitoring

## Validation Example

```rust
use validator::{Validate, ValidationError};

#[derive(Validate)]
pub struct CreateUserRequest {
    #[validate(email)]
    pub email: String,
    
    #[validate(length(min = 12, message = "Password must be at least 12 characters"))]
    pub password: String,
    
    #[validate(length(min = 1, max = 100))]
    pub username: String,
}
```

## Error Handling

```rust
pub enum ValidationError {
    InvalidType { field: String, expected: String },
    InvalidLength { field: String, min: usize, max: usize, actual: usize },
    InvalidFormat { field: String, expected: String },
    InvalidRange { field: String, min: i64, max: i64, actual: i64 },
    InvalidValue { field: String, allowed: Vec<String> },
}
```
