# Refund Policy

## Rule Definition

All refund requests must be processed according to business policy with proper validation, approval workflow, and audit logging.

## Validation Criteria

1. **Eligibility Window** — Refunds must be requested within 30 days of purchase
2. **Usage Limit** — Refunds denied if product usage exceeds 10% of quota
3. **Approval Workflow** — Refunds over $100 require manager approval
4. **Original Payment** — Refunds must go to original payment method
5. **Partial Refunds** — Partial refunds allowed with prorated calculation

## Implementation Notes

- Calculate eligibility window from payment timestamp
- Track usage metrics for quota calculation
- Implement approval workflow with notification
- Log all refund decisions for audit trail
- Send email notification on refund status

## Refund States

```
Pending → Under Review → Approved → Processing → Completed
                      → Rejected (with reason)
```

## Error Handling

```rust
pub enum RefundError {
    OutsideEligibilityWindow,
    UsageExceeded,
    AlreadyRefunded,
    ApprovalRequired,
    OriginalPaymentMethodInvalid,
    ProcessingFailed,
}
```
